    12|    12|
    13|    13|> **Yuan 的项目大脑。** 上下文会压缩，会话会结束——但项目记忆永存。

    18|    18|## ⚠️ 两层 docs 体系区分
    19|    19|
    20|    20|| 目录 | 用途 | 谁维护 | 何时读取 |
    21|    21||------|------|--------|---------|

    30|    30|## 核心原则
    31|    31|
    32|    32|1. **Agent 首次加载 = 完整继任** — 读完 Memory 后能无缝继续开发
    33|    33|2. **上下文压缩友好** — 每份文档都紧凑可扫描，不依赖大段文本
    34|    34|3. **进度驱动** — PROGRESS.md 是王，任何 Agent 先读它
    35|    35|4. **坑即知识** — PITFALLS.md 把踩坑经验结构化，是学习回环的输入
    36|    36|

    52|    52|## Agent 加载流程
    53|    53|
    54|    54|```
    55|    55|[Agent 启动]
    56|    56|    │
    57|    57|    ├── 1. read_file("docs/PROGRESS.md")
    58|    58|    │      → 获悉：当前 Phase/Stage/Task、阻塞项、下一步
    59|    59|    │
    60|    60|    ├── 2. read_file("docs/ARCHITECTURE.md")
    61|    61|    │      → 了解系统全貌
    62|    62|    │
    63|    63|    ├── 3. read_file("docs/pitfalls.md")
    64|    64|    │      → 避开已知陷阱
    65|    65|    │
    66|    66|    ├── 4. 如果当前有 Plan：
    67|    67|    │      read_file("当前会话文件夹中的 PLAN.md")
    68|    68|    │      → 读取当前 Task 的详细规格
    69|    69|    │
    70|    70|    └── 5. （可选）read_file(".yuan/docs/")
    71|    71|           → 了解最近做了什么
    72|    72|```

   106|   106|### PITFALLS.md — 踩坑知识库
   107|   107|
   108|   108|**何时更新：每次踩坑后立即记录（不要等 Phase 4）。**
   109|   109|
   110|   110|每条坑的格式：
   111|   111|```markdown
   112|   112|### PIT-NNN: [一句话标题]
   113|   113|
   114|   114|**日期:** YYYY-MM-DD
   115|   115|**类型:** [本项目 / 前端 / 后端 / DB / 部署 / 流程]
   116|   116|**严重程度:** 🔴 / 🟡 / 🟢
   117|   117|
   118|   118|**现象:** 发生了什么？
   119|   119|**原因:** 为什么？
   120|   120|**修复:** 怎么解决的？
   121|   121|**教训:** 下次怎么办？
   122|   122|
   123|   123|**归档判断:**
   124|   124|- 其他项目也会遇到？ [是/否]
   125|   125|- 有明确操作步骤？ [是/否]
   126|   126|- → [留在此文件 / 提炼为 Skill / 反馈到 Yuan]

   130|   130|- 边做边记，不要等到 Phase 4 再回忆

   132|   132|- Agent 启动时必须读 PITFALLS.md — 记了不看等于白记

   198|   198|## 与平台 Memory/Skill 工具的分工
   199|   199|
   200|   200|> 不同 Agent 平台有各自的持久化工具（如 Hermes 的 `memory`/`skill`、系统级配置等）。
   201|   201|> 本体系只管理**项目级**知识，平台级偏好留给平台自己的工具。
   202|   202|
   203|   203|| 存什么 | 放哪 |
   204|   204||--------|------|
   205|   205|| 项目进度、架构、决策、术语、踩坑 | → `docs/*.md（规格见 .yuan/docs/）`（本体系） |
   206|   206|| 用户偏好（简洁回复、语言等） | → 平台自己的 memory/配置工具 |
   207|   207|| 环境配置（OS、路径、工具版本） | → 平台自己的 memory/配置工具 |
   208|   208|| 可复用的操作流程 | → 平台自己的 skill 工具 |
