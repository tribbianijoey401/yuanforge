# PROGRESS — 进度中枢规格书

> 管辖 docs/PROGRESS.md。项目最重要的文件，回答「到哪了？下一步做什么？」

---

## 目的

Agent 每次启动必读的第一个文档。记录项目整体进度，指向当前活跃的会话文件夹。

---

---

## 状态机规则

```
[初始化]
    ↓ project-bootstrap
[Phase 1: 架构]
    ↓ Architect 产出 Plan → G1 ✓
[Phase 2: 执行]
    ↓ Conductor 派发 → G2 ✓
[Phase 3: 收尾]
    ↓ G3/G4 ✓
[Phase 4: 回顾]
    ↓ Conductor 归档
[已交付]
```

### 字段更新规则
- **当前 Phase**: Conductor 在 Phase 切换时更新
- **当前会话**: 每个新会话开始时 Conductor 更新
- **功能清单**: 会话完成时追加
- **历史会话**: 会话结束时 Conductor 追加

---

## 生命周期

| 阶段 | 操作 | 执行者 |
|------|------|--------|
| 项目初始化 | 从模板创建，状态=初始化 | project-bootstrap |
| Phase 切换 | 更新当前 Phase | Conductor |
| 每个会话 | 更新当前会话、功能清单、历史会话 | Conductor |
| Phase 4 | 归档总结 | Conductor |
