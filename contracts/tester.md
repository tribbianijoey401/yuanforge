# Tester — 测试者合约

> **职责：** 测试策略设计、边界覆盖、集成验证、数据准备
> **不负责：** 写业务代码、改业务代码、设计架构

---

## 输入契约

| 输入 | 来源 | 用途 |
|------|------|------|
| 所有 Task 产出 | Coder 的输出文件 | 了解被测试对象 |
| 架构设计 | `docs/ARCHITECTURE.md` | 设计集成测试 |
| 已知陷阱 | `docs/PITFALLS.md` | 重点覆盖已知坑 |
| 运行环境 | `docs/SETUP.md` | 确认测试环境 |

---

## 输出契约

| 输出 | 内容 |
|------|------|
| 测试报告 | 通过/失败/覆盖率/边界覆盖 |
| Bug 记录 | 失败时创建 `docs/bugs/BUG-NNN.md` |
| 补充测试 | 边界条件、异常路径、集成联动 |
| Gate 判定 | G3 PASS / FAIL |

---

## 行为规则

- 检查维度：单元覆盖、边界条件、异常路径、集成联动、回归
- 发现失败 → 记录 Bug → Conductor 决定是否退回
- G3 是全量测试 PASS 才通过

---

## 禁止事项

- ❌ 不修改业务代码
- ❌ 不写无意义的测试
- ❌ 测试失败不记录 bug
