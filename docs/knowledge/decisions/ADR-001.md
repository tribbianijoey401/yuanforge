# ADR-001: Goal/Workflow Protocol 统一 Gate 模型 + goal-protocol.md 删除

## Context

N13 缺陷暴露了 `goal-protocol.md` 和 `workflow-protocol.md` 之间的命名冲突。
深入分析发现：
- Human Gate 被错误地定义在意图层文件里，本质是控制流闸门，应该归 workflow
- 质量闸 G1-G4 扎根于 6 个活跃文件，命名空间大；Human Gate G1-G4 只活在待删文件中
- 第一性原理结论：框架 irreducibly 需要"流程程序"和"意图/状态模型"两份文件，但当前形态不是最小必要形式

## Decision

1. **删除 `goal-protocol.md`**，其内容拆分归位：
   - Goal YAML 结构 / Goal Stack → `object-protocol.md`（数据模型）
   - Human Gate HG1-HG4 → `workflow-protocol.md §十`（控制流，与质量闸同文件自然隔离）

2. **Human Gate 改名为 HG1–HG4**（而非质量闸改名 QG）：
   - 质量闸 G 轴扎根于 6 个活跃文件 → 改名成本 6 文件
   - Human Gate G 轴只活在 goal-protocol.md（待删）→ 改名成本 2 文件
   - 命名较小轴，爆炸半径从 6 文件降到 2 文件

3. **在 `workflow-protocol.md` 新增「Gate 模型」节**，统一定义 Gate 原语：
   ```
   Gate = { kind: quality|human, id, trigger, pass_condition, fail_action }
   ```
   质量闸沿用 G1–G4（不改动现有 6 文件），Human Gate 用 HG1–HG4

4. **Loop 三处去重**：`AGENTS.md` 和 `conductor.md` 的调度循环降级为"见 workflow-protocol.md"指针

## Consequences

- 文件数 -1（goal-protocol.md 消失）
- 消除 Human Gate 跨文件定义（根因修复）
- 质量闸 6 文件零改动
- 统一 Gate 原语，消除 ad-hoc 闸门
- conductor.md 保留跨会话恢复/升级路由/Human Gate 执行的角色专属段
- AGENTS.md 保留一行压缩速览 + 指针（维持平台无关入口的自洽 orientation）
